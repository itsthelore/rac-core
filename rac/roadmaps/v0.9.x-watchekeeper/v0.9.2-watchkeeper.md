# Roadmap: v0.9.2 — Watchkeeper

## Status

Proposed

## Summary

Introduce Watchkeeper as RAC's product knowledge review surface.

Watchkeeper consumes existing RAC intelligence and presents it as a unified review experience for product knowledge changes.

## Problem

Repository review and intent analysis provide valuable intelligence, but reviewers require a consolidated experience that answers:

> What changed, and does this change need attention?

Watchkeeper provides this capability.

## Goals

- Introduce RAC Watchkeeper.
- Consolidate review intelligence.
- Produce reviewer-focused outputs.
- Support machine-readable integrations.
- Establish the foundation for CI workflows.

## Scope

### Watchkeeper Command

Provide:

```bash
rac watchkeeper
```

### Review Report

Watchkeeper shall present:

```text
Changed Artifacts

Validation Status

Relationship Impact

Repository Changes

Intent Findings

Review Recommendations
```

Example:

```text
RAC Watchkeeper

Product knowledge changes detected.

Changed:

+ 3 Requirements
~ 1 Decision

Validation:

✓ All artifacts valid

Relationships:

⚠ Missing ADR relationship

Findings:

Specificity regression detected.

Review Recommended.
```

### Review Recommendations

Provide deterministic recommendations for human review.

Examples:

```text
Review Recommended

Reasons:

- Acceptance criteria removed
- Broken relationship
- Specificity regression detected
```

Watchkeeper does not determine whether a change is correct.

Watchkeeper identifies changes requiring attention.

### Structured Output

Provide machine-readable output.

```bash
rac watchkeeper --format json
```

Consumers include:

- GitHub Actions
- CI systems
- MCP servers
- Explorer
- Atlas
- AI agents

### GitHub Output Format

Provide GitHub-oriented rendering.

```bash
rac watchkeeper --format github
```

This output will be consumed by future GitHub workflows.

## Deliverables

### Commands

```bash
rac watchkeeper

rac watchkeeper --format json

rac watchkeeper --format github
```

### Services

```text
WatchkeeperService
```

### Outputs

- Human-readable reports
- JSON reports
- GitHub-formatted reports

## Non-Goals

This initiative shall not:

- implement independent intelligence logic
- replace product reviewers
- approve changes automatically
- require GitHub
- require hosted infrastructure

## Acceptance Criteria

A user can:

1. Run Watchkeeper.
2. Review product knowledge changes.
3. Understand intent impact.
4. View relationship impact.
5. Receive review recommendations.
6. Consume machine-readable outputs.

through a single review surface.

## Related Requirements

- Product Intent CI (Watchkeeper)

## Related Decisions

- ADR-015 Explorer as a Consumer

## Success Measures

Success is achieved when Watchkeeper becomes the primary review surface for product knowledge changes while remaining a consumer of RAC Core intelligence.
