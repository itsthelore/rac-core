# Roadmap: v0.9.1 — Intent Analysis

## Status

Proposed

## Summary

Introduce deterministic product intent analysis capabilities that identify meaningful changes within product knowledge artifacts.

This initiative transforms repository review from:

> What changed?

to:

> What changed, and why might it matter?

## Problem

Raw file diffs are often insufficient for reviewing product knowledge.

Reviewers care less about Markdown changes and more about:

- requirements becoming less precise
- constraints being weakened
- acceptance criteria being removed
- ambiguity being introduced
- relationships being impacted

RAC should provide deterministic analysis of these changes.

## Goals

- Introduce intent-aware change analysis.
- Detect reductions in product clarity.
- Detect weakened requirements.
- Detect ambiguous language.
- Surface relationship impact.
- Produce actionable review findings.

## Scope

### Product Intent Diffing

Translate artifact changes into product-impact summaries.

Examples:

```text
Acceptance criteria removed.

Success measures modified.

Constraint weakened.

Requirement expanded.
```

### Specificity Regression Detection

Detect measurable requirements becoming vague.

Example:

```diff
- Upload must complete within 5 seconds.
+ Upload should complete quickly.
```

Finding:

```text
Specificity regression detected.
```

### Ambiguity Detection

Detect introduction of ambiguous language.

Examples:

- easy
- intuitive
- simple
- seamless
- user-friendly
- scalable

### Constraint Change Detection

Detect:

- removed constraints
- weakened constraints
- mandatory language reductions

Examples:

```diff
- must
+ should
```

### Relationship Impact Analysis

Identify relationship changes and downstream impact.

Examples:

```text
REQ-010 modified.

Affected:

- ADR-004
- ROADMAP-Q3
```

### Scope Linkage Analysis

Identify newly introduced scope lacking supporting context.

Examples:

- requirement without roadmap
- dependency without ADR
- design behaviour without requirement

## User Experience

CLI:

```bash
rac review --intent
```

## Deliverables

### Services

```text
IntentAnalysisService
```

### Outputs

Structured intent findings.

## Non-Goals

This initiative shall not:

- generate review comments
- create GitHub integrations
- enforce governance policies
- automatically approve or reject changes

## Acceptance Criteria

A user can:

1. Compare repository changes.
2. Understand product impact.
3. Detect weakened requirements.
4. Detect ambiguity.
5. Detect relationship impact.

without manually inspecting Markdown diffs.

## Related Requirements

- Product Intent CI (Watchkeeper)

## Success Measures

Success is achieved when RAC can explain meaningful product knowledge changes using deterministic analysis rather than raw file differences.
