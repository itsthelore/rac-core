# Roadmap: v0.9.0 — Repository Review

## Status

Proposed

## Summary

Introduce repository review capabilities that allow RAC to compare two repository states and identify product knowledge changes.

This initiative establishes the foundational review intelligence required by Watchkeeper.

The goal is not yet to determine whether changes are good or bad.

The goal is to determine what changed.

## Problem

RAC currently provides intelligence about the current state of a repository.

Watchkeeper requires intelligence about changes between repository states.

Before RAC can provide pull request reviews, it must be capable of comparing:

- a base revision
- a head revision

and producing a structured understanding of repository changes.

## Goals

- Introduce repository comparison capabilities.
- Identify changed product artifacts.
- Identify validation changes.
- Identify relationship changes.
- Identify repository statistics changes.
- Provide a reusable review model for future consumers.

## Scope

### Repository Review Service

Implement a repository review service capable of comparing two repository states.

Inputs:

```text
Base Repository
Head Repository
```

Outputs:

```text
Changed Artifacts
Validation Delta
Relationship Delta
Statistics Delta
```

### Changed Artifact Detection

Detect:

- Added artifacts
- Modified artifacts
- Removed artifacts

Across:

- Requirements
- Decisions
- Roadmaps
- Designs
- Prompts

### Validation Delta

Compare validation outcomes between repository states.

Examples:

```text
Invalid Artifacts
0 → 1
```

### Relationship Delta

Compare relationship integrity between repository states.

Examples:

```text
Broken Relationships
0 → 2
```

### Statistics Delta

Compare repository-level statistics.

Examples:

```text
Requirements
42 → 45

Decisions
12 → 13
```

```bash
rac review
```

### Commands

```bash
rac review
```

### Services


### Outputs

## Non-Goals

This initiative shall not:

- perform intent analysis
- classify review findings
- recommend human review
- integrate with GitHub
- implement Watchkeeper

## Acceptance Criteria

A user can:

1. Compare repository states.
2. Identify changed artifacts.
3. View validation deltas.
4. View relationship deltas.
5. View repository statistics deltas.

through a single repository review capability.

## Success Measures

Success is achieved when repository change intelligence becomes available through RAC Core and can be consumed by future Watchkeeper capabilities.
