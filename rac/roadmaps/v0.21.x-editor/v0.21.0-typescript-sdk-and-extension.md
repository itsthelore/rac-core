# RAC v0.21.0 — TypeScript SDK and Extension MVP

## Status

Planned

## Context

The Python SDK (v0.20.0) made RAC importable as a library, but the surface where
a human authors decisions and requirements — the editor — has no RAC feedback.
This release opens the `v0.21.x` series by delivering two things: a thin
TypeScript client for RAC, and a VS Code / Cursor extension built on it that
validates artifacts in-editor.

Per ADR-063 the client is a thin client: it shells out to the installed `rac`
binary and deserializes the stable `--json` contracts (ADR-007); it reimplements
none of the engine, so it always agrees with the CLI. This is the Ruff / ESLint
editor-extension pattern, and it is the foundation the rest of the `v0.21.x`
series (cross-artifact enforcement, authoring, navigation, awareness,
visualization, robustness) builds on.

## Outcomes

- A published npm package wraps the `rac` CLI with typed results and one error
  root, usable by any Node consumer (the extension is its first).
- A VS Code / Cursor extension shows validation findings inline — live as you
  type, and on open/save — identical to what `rac validate` reports.
- Hovering an artifact ID/alias resolves it; go-to-definition jumps to the target.
- A missing `rac` binary surfaces a one-time install prompt, never a wall of errors.
- A CI workflow builds and tests both packages on a Node toolchain.

## Initiatives

### Initiative 1 — Thin TypeScript client (`@rac/sdk`)

A zero-runtime-dependency Node package: `RacClient` over `rac … --json`
(`validateFile`, `validateText` via stdin, `validateDirectory`, `resolve`,
`find`, `validateRelationships`, `review`, `stats`, `exportCorpus`, `version`),
a `RacError` hierarchy (a validation failure is a result, not an exception), and
an injectable runner seam so the client is unit-testable without a binary.

### Initiative 2 — VS Code / Cursor extension MVP

Live validation diagnostics (the unsaved buffer piped through `rac validate -`,
debounced; plus open/save), artifact-ID hover and go-to-definition via
`rac resolve`, `rac` discovery (setting → `RAC_BIN` → `PATH`), and graceful
handling when `rac` is absent. Only Markdown with `schema_version` frontmatter is
treated as a RAC artifact.

### Initiative 3 — TypeScript CI

A workflow that installs `rac`, builds and tests the SDK (including real-`rac`
integration tests), then typechecks and bundles the extension. Mirrors the CI
trigger policy (ADR-027).

## Constraints

- Thin client only (ADR-063): no native reimplementation of parse/classify/validate.
- Consume only stable contracts — `--json`, exit codes (ADR-007).
- Desktop editors (subprocess); web VS Code out of scope.
- The extension requires `rac` present; it never bundles an engine.

## Non-Goals

- A native in-process TS engine or browser execution (ADR-063).
- Cross-artifact enforcement, authoring aids, navigation beyond hover/definition,
  awareness, visualization, and packaging — those are v0.21.1–v0.21.6.
- An LSP server.

## Implementation Contract

- `@rac/sdk` exposes `RacClient` with the methods above and a `RacError` root;
  result types mirror the `rac --json` / `export` shapes.
- The extension renders `rac validate` findings as diagnostics for the active
  buffer, and backs hover/definition with `rac resolve`.
- CI builds the SDK before the extension (which consumes it via `file:../rac-sdk`).

## Success Measures

- `from "@rac/sdk"` imports resolve; the SDK's unit and real-`rac` integration
  tests pass.
- Editing an invalid artifact shows the same findings as `rac validate` on it.
- Hovering an artifact ID resolves it; go-to-definition opens the target.
- With `rac` absent, the extension prompts to install and shows no errors.

## Risks

- `rac` missing or version-skewed. Mitigated by discovery order and a graceful
  prompt (deeper skew handling lands in v0.21.6).
- Subprocess latency per keystroke. Mitigated by debouncing and validating the
  single active buffer, not the corpus.

## Assumptions

- The `rac … --json` contracts are stable enough (ADR-007) to be the SDK surface.
- Target users have, or can `pip install`, `rac`.
- The repository restructure settles the `typescript/` home before release.
