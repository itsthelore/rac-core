# RAC v0.21.12 — Graph View Polish

## Status

Planned

## Context

v0.21.8 shipped the corpus graph view; this release closes the gap to
Obsidian-grade feel and accessibility. Force-strength controls, keyboard
navigation, cleaner orphan handling, and a sensible local-graph default move the
graph from a demo surface toward a daily instrument. It must land on the v0.21.9
test gate — polishing an untested renderer is exactly how the v0.21.8 layout
regression happened.

## Outcomes

- The graph is tunable, keyboard-accessible, and handles orphans gracefully.
- The local graph is a first-class default rooted on the active artifact.

## Initiatives

### Initiative 1 — Force controls

User-adjustable centre / repel / link force and link distance (Obsidian-style)
with good defaults, while the seeded determinism of the layout is preserved.

### Initiative 2 — Accessibility

Keyboard navigation of nodes and edges with a focus model, so the graph is not
mouse-only.

### Initiative 3 — Orphans & defaults

Clearer orphan placement (no frame-edge piling, an artifact of the v0.21.8
containment fix) and an opt-in local-graph default on the active artifact.

## Constraints

- Thin client (ADR-063): no positions in the export; layout stays client-side
  and seeded (ADR-007).
- Changes land behind the v0.21.9 viewer test gate.
- Offline; no new dependencies that break the single-file Portal.

## Non-Goals

- A new graph engine, persisted layouts, or 3D.

## Implementation Contract

- Force sliders with deterministic defaults, keyboard node navigation, orphan
  handling that avoids frame-edge piling, and an opt-in local-graph default —
  all covered by viewer tests (v0.21.9).

## Success Measures

- Force controls change the layout live and remain deterministic per seed.
- The graph is navigable by keyboard.
- Orphans no longer pile at the frame edge, and a determinism snapshot guards
  the layout.

## Risks

- More graph behaviour on a renderer that recently regressed. Mitigated by the
  v0.21.9 tests.
- Accessibility on node-link layouts is non-trivial. Scoped to focus + arrow
  traversal.

## Assumptions

- The seeded force layout (v0.21.8) extends to tunable forces without losing
  determinism.
