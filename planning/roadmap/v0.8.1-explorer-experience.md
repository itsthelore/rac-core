# RAC v0.8.1 — Workspace Experience

## Problem

v0.8.0 introduces Explorer as a terminal-native repository workspace.

Users can:

- Browse artifacts
- Search repositories
- Inspect knowledge
- Navigate relationships
- View repository issues

However, Explorer still behaves primarily as a repository viewer.

Modern developer tools such as:

- Claude Code
- Amp
- Cursor
- VS Code
- Obsidian

provide onboarding, workspace continuity, and workflow optimization that reduce friction and encourage habitual use.

Explorer requires a stronger workspace experience that improves:

- Adoption
- Retention
- Discoverability
- Daily usability

without introducing repository hosting, cloud services, or AI assistants.

---

## Vision

Explorer shall evolve from a repository explorer into a repository workspace.

Users should feel that they are entering a familiar environment rather than launching a command.

Explorer should remember context, reduce navigation friction, and help users quickly resume work.

---

## Outcomes

Users should be able to:

```text
Open Explorer

Resume where they left off

Locate repositories quickly

Understand how to navigate

Discover capabilities naturally
```

with minimal onboarding effort.

---

# Initiatives

## Initiative 1 — First Run Experience

Provide onboarding for first-time users.

Example:

```text
Welcome to RAC Explorer

Repository Knowledge Workspace

Press / to search

Navigate with ↑ ↓

Press Enter to open

Press q to quit
```

The onboarding experience shall:

- Introduce core navigation
- Introduce search
- Explain repository concepts
- Explain artifact relationships

Onboarding should require less than 30 seconds.

---

## Initiative 2 — Repository Selection

Support repository discovery and selection.

Example:

```text
Select Repository

> requirements-as-code
  payments-platform
  identity-service

Recent

  analytics-platform
```

Explorer shall support:

- Current directory repositories
- Recently opened repositories
- Manual path selection

---

## Initiative 3 — Recent Repositories

Maintain a list of recently opened repositories.

Example:

```text
Recent Repositories

requirements-as-code
payments-platform
identity-service
```

Recent repositories should be available from startup.

---

## Initiative 4 — Workspace Persistence

Persist workspace state between sessions.

Explorer may restore:

- Last viewed artifact
- Last active repository
- Last selected view
- Last search query

Users should be able to resume work quickly.

---

## Initiative 5 — Navigation Enhancements

Improve navigation efficiency.

Examples:

```text
Back

Forward

Recently Viewed

Favorites
```

Explorer should minimize repetitive navigation.

---

## Initiative 6 — Keyboard Shortcut System

Provide discoverable keyboard shortcuts.

Example footer:

```text
/ Search

h Home

b Back

r Relationships

q Quit
```

Shortcuts shall remain visible throughout the interface.

---

## Initiative 7 — User Preferences

Provide lightweight customization.

Examples:

- Theme
- Default landing view
- Artifact grouping mode
- Startup behavior

Preferences shall remain optional.

---

## Initiative 8 — Knowledge Domains

Introduce repository domain navigation.

Explorer may group artifacts into logical domains.

Example:

```text
Payments

Identity

Reporting

Analytics
```

Domain navigation shall remain viewer-only.

Domain intelligence shall originate from RAC Core.

---

# Requirements

### REQ-001 — Workspace Continuity

Explorer shall persist sufficient state to allow users to resume work.

### REQ-002 — Repository Discovery

Explorer shall support repository discovery and selection.

### REQ-003 — Recent Repository Tracking

Explorer shall maintain a list of recently opened repositories.

### REQ-004 — Discoverable Navigation

Explorer shall expose visible keyboard shortcuts.

### REQ-005 — Optional Personalization

Explorer shall support optional user preferences.

### REQ-006 — Consumer Architecture

Workspace features shall not introduce independent repository intelligence.

---

## Non-Goals

- Repository synchronization
- Cloud accounts
- Multi-user collaboration
- Artifact editing
- AI assistants

---

## Success Measures

Users regularly launch:

```bash
rac explorer
```

without supplying repository paths.

Explorer remembers user context between sessions.

New users successfully navigate repositories without external documentation.

Workspace continuity reduces time-to-context after startup.
