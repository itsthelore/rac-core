# RAC v0.8.0 — Explorer Foundation

## Problem

RAC repositories accumulate structured product knowledge over time.

As repositories grow, users increasingly need to answer questions such as:

```text
What are we building?

Why was this decision made?

What is connected?

What needs attention?

How healthy is the repository?
```

RAC currently exposes repository intelligence through individual commands.

While powerful and scriptable, command-oriented workflows become increasingly difficult as repositories scale.

Users must remember:

* Which command to execute
* Which artifact to inspect
* Which output to interpret
* How repository knowledge is connected

A terminal-native exploration experience is required.

---

## Vision

Explorer shall provide a unified workspace for understanding repository knowledge.

Explorer is not a replacement for RAC commands.

Explorer is a consumer of RAC capabilities that provides:

* Navigation
* Discovery
* Context
* Relationships

from a single entry point.

Explorer shall feel closer to:

* Claude Code
* Amp
* Lazygit
* k9s

than to a traditional dashboard.

Explorer shall remain compliant with ADR-015.

---

## Outcomes

Users should be able to:

```text
Find knowledge
Inspect artifacts
Traverse relationships
Understand repository health
```

without needing to memorize RAC commands.

# Initiatives

## Initiative 1 — Repository Home

Provide a repository landing experience.

Example:

```text
RAC Explorer

Health Score: 87%

Attention

REQ-041 missing metrics
ADR-017 orphaned

Recent Activity

ADR-022 accepted
REQ-041 updated
```

The home screen should immediately communicate repository status.

---

## Initiative 2 — Global Command Bar

Provide a globally accessible command and search interface.

Activation:

```text
/
```

Example:

```text
/

req-041

payments

health

orphaned decisions
```

The command bar shall become the primary navigation mechanism.

Supported results:

* Artifacts
* Views
* Relationships
* Actions

---

## Initiative 3 — Artifact Navigation

Provide keyboard-driven artifact navigation.

Users shall be able to:

* Browse artifacts
* Open artifacts
* Return to previous views

Navigation shall remain performant for repositories containing at least 1,000 artifacts.

---

## Initiative 4 — Contextual Inspection

Display repository intelligence alongside artifact content.

Example:

```text
REQ-041

Confidence: 92%

Sections

✓ Problem
✓ Requirements
✗ Risks

Relationships

ADR-012
ROADMAP-Q3
```

Data shall be sourced from existing RAC services.

---

## Initiative 5 — Relationship Navigation

Relationships shall be first-class navigation elements.

Users shall be able to move between connected artifacts.

Explorer shall make repository structure discoverable without graph visualizations.

---

## Initiative 6 — Attention View

Provide visibility into repository issues.

Examples:

* Missing sections
* Broken references
* Orphaned artifacts
* Unknown artifacts

Users shall be able to navigate directly to affected artifacts.

---

# Requirements

### REQ-001 — Terminal Native

Explorer shall operate entirely within a terminal environment.

### REQ-002 — Consumer Architecture

Explorer shall consume RAC services.

Repository intelligence shall not exist exclusively within Explorer.

### REQ-003 — Command Parity

Capabilities exposed through Explorer shall remain accessible through CLI commands.

### REQ-004 — Repository Local

Explorer shall operate on local repositories.

### REQ-005 — Keyboard First

All functionality shall be accessible without a mouse.

### REQ-006 — Responsive Navigation

Navigation shall remain performant for repositories containing at least 1,000 artifacts.

### REQ-007 — Viewer Agnostic Compliance

Explorer shall comply with ADR-014.

### REQ-008 — Discoverability

Explorer shall expose visible navigation hints and keyboard shortcuts.

### REQ-009 — Search Centric

Explorer shall provide a globally accessible command bar.

Default activation:

```text
/
```

### REQ-010 — Service Reuse

Explorer shall consume existing RAC services.

Explorer shall not duplicate business logic.

---

## Non-Goals

* Artifact editing
* AI assistants
* Repository hosting
* Web applications
* Project management workflows

---

## Success Measures

Users adopt:

```bash
rac explorer
```

as the preferred interactive entry point into RAC repositories.

CLI commands remain the preferred automation interface.

Repository intelligence remains owned by RAC Core.
