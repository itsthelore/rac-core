# RAC v0.8.2 — Knowledge Operations

## Problem

v0.8.0 provides repository exploration.

v0.8.1 provides a persistent workspace experience.

Users can understand repository knowledge effectively, but operational workflows still require direct CLI usage.

Common workflows include:

```text
Import documents

Validate repositories

Review repository health

Inspect gaps

Improve artifacts

Analyze coverage
```

These workflows are currently command-oriented.

Explorer should provide operational visibility and orchestration while remaining compliant with ADR-015.

---

## Vision

Explorer shall become the primary operational interface for maintaining repository knowledge.

Repository intelligence shall continue to reside within RAC Core.

Explorer shall orchestrate existing capabilities rather than reimplement them.

---

## Outcomes

Users should be able to:

```text
Import knowledge

Identify repository issues

Run repository analysis

Review coverage

Improve repository quality
```

without memorizing RAC commands.

---

# Initiatives

## Initiative 1 — Document Import Workspace

Provide an interactive ingestion workflow.

Example:

```text
Import Knowledge

Select File

> PRD.pdf

Detected

Requirements: 17
Decisions: 4
Roadmaps: 2
```

Explorer shall consume RAC ingestion services.

---

## Initiative 2 — Import Review

Provide review workflows prior to ingestion.

Example:

```text
Detected Requirements

REQ-CANDIDATE-1
REQ-CANDIDATE-2

Detected Decisions

ADR-CANDIDATE-1
```

Users shall be able to review proposed imports before committing them.

---

## Initiative 3 — Portfolio Insights

Provide repository-wide health visibility.

Example:

```text
Coverage Score      87%

Requirements        42
Decisions           17

Broken References    3
Unknown Artifacts    1
```

Metrics shall be sourced from RAC Core.

---

## Initiative 4 — Validation Center

Provide visibility into repository validation.

Example:

Warnings: 12

Users shall be able to navigate directly to affected artifacts.

# Requirements

### REQ-001 — Service Reuse

Operational workflows shall consume existing RAC services.

### REQ-002 — Ingestion Integration

Explorer shall provide workflows for RAC ingestion capabilities.

### REQ-003 — Validation Integration

Explorer shall provide workflows for RAC validation capabilities.

### REQ-004 — Portfolio Integration

Explorer shall surface portfolio intelligence.

### REQ-005 — Improvement Integration

Explorer shall surface repository improvement recommendations.

### REQ-006 — Review Before Import

Users shall be able to review detected artifacts prior to ingestion.

### REQ-007 — Operational Discoverability

Operational workflows shall be accessible through search and navigation.

### REQ-008 — Consumer Architecture

Explorer shall not become a source of repository intelligence.

---

## Non-Goals

- Artifact editing
- AI-generated repository changes
- Automatic repository modification
- Cloud execution
- Background services

---

## Success Measures

Users perform routine repository maintenance through Explorer.

Explorer becomes the preferred operational interface for repository quality management.

Repository intelligence remains implemented within RAC Core.

Operational workflows remain accessible through standard CLI commands.

Explorer continues to act as a consumer rather than a source of repository intelligence.
