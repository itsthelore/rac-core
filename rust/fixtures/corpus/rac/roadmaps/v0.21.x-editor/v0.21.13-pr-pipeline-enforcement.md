# RAC v0.21.13 — Enforcement in the PR Pipeline

## Status

Achieved

## Context

The editor enforces locally, but an IDE warning is a suggestion a developer — or
an AI agent operating outside the editor — can ignore. "Enforcement is the
product" (ADR-049) only compounds when it is binding: at the merge gate, agent-
agnostic and org-wide. This release carries `rac validate` +
`rac relationships --validate` + `rac review` into a required PR check,
sequenced engine-first so the Action stays a thin consumer of a stable contract
rather than re-deriving anything.

## Outcomes

- Every PR runs the same RAC checks and annotates findings inline.
- Corpus integrity becomes a required, non-bypassable merge gate.
- The Action reimplements no engine logic.

## Initiatives

### Initiative 1 — A CI-shaped output contract (engine-first)

An additive `rac review --pr` (and/or SARIF) with stable, machine-gradable
findings and exit codes, so any CI consumer — this Action and future ones —
inherits one authoritative output.

### Initiative 2 — A thin GitHub Action

An Action that invokes the contract and surfaces findings as inline annotations
and a required status check.

### Initiative 3 — Policy hook

Which finding classes are blocking versus advisory is read from the corpus, not
hardcoded — aligning with the policy-as-config in v0.21.14.

## Constraints

- Thin consumer (ADR-063): the Action consumes `rac` output and computes
  nothing.
- Additive contract (ADR-007); deterministic findings and exit codes.

## Non-Goals

- A hosted service, a bypass mechanism, or changing how `rac` computes
  relationships.

## Implementation Contract

- `rac` emits a machine-gradable PR/SARIF findings contract with stable exit
  codes.
- A GitHub Action runs it on PRs as a required check with inline annotations.
- Blocking-versus-advisory is corpus-configured.

## Success Measures

- A PR introducing a broken or retired-decision reference fails the check with
  an inline annotation matching `rac relationships --validate`.
- The gate is required and reproduces local findings exactly.

## Risks

- A new CLI contract is real engine work. Mitigated by keeping it additive and
  SARIF-shaped.
- Relationship findings lack line anchors. Mitigated by anchoring to the
  reference token, as the editor already does.

## Assumptions

- `rac validate` / `relationships` / `review` expose enough to produce SARIF/PR
  output additively.
